Sync

The Sync plugin is for pushing text messages received on FrontlineSMS to a website


NOTE: To use Sync, you must be running FrontlineSMS version 1.7.00 or above. To find out, check the top left of the screen when running FrontlineSMS. The number that appears is the version you are running. If you need an upgrade and haven't received an email invitation to do so, get in touch via the Contact form on the website

What is Sync?

The Sync plugin is a tool which can be used to pushing messages to website/webserver e.g. an Ushahidi instance. It is built for situations where where a FrontlineSMS user would like to sync received messages with a publicly accessible website. For purposes of synchronisation, Sync requires an Internet connection but if this is not available, it queues up the messages and pushes them to the target URL once connectivity is restored. Usually, the target website will have a URL format in which is used fo receiving messages being pushed to it. For example: http://example.com/messages?key=kDE48Aq&s={sender_number}&m={message_content} where: {sender_number} is the phone number of the person sending the message and {message_content} is the body of the text message.

The URL to which messages are being pushed to is shown in [2].

The Synchronization History [1] is a log of all the sync activity taking place (Messages are only pushed to the sync URL only after they have been saved in the FrontlineSMS database) and [4] is used to clear the synchronisation log.

The Synchornization URL [2] is the URL to which the messages will be pushed to and [1] is the list of parameters that shall be fed to the Synchronisation URL. The parameter list can be modified by using the Add [3] and Delete [4] buttons to create and remove parameters respectively.

Clicking the Add button above on the Sync plugin settings window above brings up the following dialog;

Key is the name of the parameter on the target website. Therefore, if the URL for receiving text messages on the target website is http://example.com/sms/?key=4595DFJ&s={sender_number}&m={message}, the value of key could be "key", "s" or "m" while Value could be "4595DFJ", {sender_number} or {message}.

During synchronisation, Sync re-constructs the synchornisation URL to include these parameters and substitutes all items in "{}" with the actual values of the message
